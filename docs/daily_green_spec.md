# Daily Green プロダクト定義書

## 1. プロダクトコンセプト

**「デッドタイム（期限）」で後回しを卒業する。**
習慣化に「締め切り」という緊張感と、GitHubのような「積み上げの可視化」を組み合わせることで、ユーザーの実行力を最大化する習慣管理アプリ。

### ターゲットユーザー

* ついつい物事を後回しにしてしまう人

* 習慣化アプリが続かなかった人（目標が漠然としすぎていた人）

* 自分の努力を視覚的に「草（ログ）」として眺めるのが好きな人

## 2. コア機能

### ① Activity Log（活動ログ）

* GitHubのコントリビューショングラフ形式で、日々の達成度を可視化。

* 達成数やデッドタイムの遵守状況に応じて緑の濃淡が変化。

### ② デッドタイム（期限）管理

* 各習慣に「何時までにやるか」というデッドタイムを設定。

* 次のデッドタイムまでのカウントダウンを表示し、適度なプレッシャーを与える。

* 期限を過ぎた場合は「遅延」として記録。

### ③ 習慣管理（今日の習慣）

* ステータス：未完了 / 完了 / 遅延完了 / 未達成。

* 完了ボタン押下時に達成感を演出するUI。

### ④ ソーシャルシェア

* ロググラフと継続日数をSNS（Xなど）に最適化された形式で画像生成・シェア。

* 継続を周囲に宣言することで、モチベーションを維持。

## 3. 用語定義

| **用語**         | **定義**                                         |
| :--------------- | :----------------------------------------------- |
| **デッドタイム** | その習慣を完了させるべき最終期限時刻。           |
| **草（ログ）**   | 日々の活動を正方形のタイルで表したもの。         |
| **ストリーク**   | 途切れずに習慣を継続している日数。               |
| **遅延完了**     | デッドタイムを過ぎてから完了ボタンを押した状態。 |

## 4. UI/UX 仕様案（画像からの抽出）

### ホーム画面（ライト/ダーク）

* **ヘッダー:** アプリ名、ストリーク数、設定アイコン。

* **メイン:** Activity Log（横長のヒートマップ）。

* **サブ:** 次のデッドタイムまでのタイマー（カード形式）。

* **リスト:** 今日実行すべき習慣のカードリスト。

### 習慣カードの構成

* 絵文字アイコン + 習慣名。

* デッドタイムの表示。

* アクションボタン（Done / Done(遅延)）。

* チェックマーク（完了済みの場合）。

## 5. データ構造（案）

エンジニアが実装を始める際に必要となるデータ項目です。

### Habit（習慣）

* `id`: UUID

* `name`: string（習慣名）

* `emoji`: string（アイコン用）

* `dead_time`: time（期限時刻）

* `order`: int（表示順）

### DailyRecord（日々の記録）

* `habit_id`: UUID

* `date`: date（日付）

* `status`: enum (uncompleted, done, done_late, missed)

* `completed_at`: datetime（実際に完了した時間）

## 6. 今後の検討事項（Q&A）

* **Q. 遅延完了の扱い:** ログの色はどうする？（例：黄色や薄緑にするなど）

* **Q. 複数デッドタイム:** 同じ習慣を1日に複数回行う場合はどう管理するか？

* **Q. 通知仕様:** デッドタイム直前以外に、朝のやる気出し通知などは必要か？

* **Q. ゲーミフィケーション:** 連続達成でレベルが上がる、植物が育つなどの要素は入れるか？
